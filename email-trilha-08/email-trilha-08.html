Quer menos retrabalho no código?
Não comece pelo código.

Valide a lógica antes de mexer nos arquivos.

Antes de pedir código, peça um plano.

Ao usar o Claude Code, pular direto para a implementação pode trazer escolhas que não combinam com o seu projeto.

Peça para ele mapear o caminho. Assim você valida a lógica antes de qualquer alteração.

✓ Teste este pedido:

“Analise este projeto e me apresente um plano para implementar [funcionalidade]. Diga quais arquivos serão alterados, os riscos e como você pretende testar. Não implemente ainda.”

O resumo: analisar → planejar → implementar. Código rápido é bom. Código com contexto é melhor.
